Kenapa Kamu (Mungkin) Suka Markdown?

August 3, 2013

Markdown adalah markup language yang inisiasinya dilakukan oleh John Gruber, dan Aaron Swartz juga turut andil dalam membentuk tagsnya. Markdown bukan pengganti dari HTML, jadi kita bisa bikin garis besarnya disitu. Markdown, untuk saya, lebih menekankan di kemudahan membacanya tanpa harus belepotan membaca teks dengan tags HTML.

Markdown is intended to be as easy-to-read and easy-to-write as is feasible. -John Gruber

Contoh Markdown dan HTML

Heading, di dunia HTML terbagi menjadi 6 turunan, mulai dari H1 sampai dengan H6, walaupun yang sering digunakan adalah H1 sampai dengan H3. Tulisan tebal, miring, dan link memiliki tags yang berbeda di HTML dan Markdown. Tidak berbeda dengan HTML, Markdown juga membutuhkan tags untuk mendefinisikan Heading, tulisan, link maupun image. Berikut contoh tags yang digunakan di Markdown dan HTML.

  1. Hi, Dunia.

    HTML : <h1>Hi, Dunia</h1>.

    Markdown: #Hi, Dunia..

  2. Tulisan Tebal

    HTML : <b>Tulisan Tebal</b>

    Markdown : **Tulisan Tebal**

  3. Tulisan Miring

    HTML : <b>Tulisan Miring_</b>

    Markdown : _Tulisan Miring_

  4. Referensi Tautan

    HTML : <a href="http://pwwdp.com">Referensi Tautan</a>

    Markdown : [Referensi Tautan](http://pwwdp.com)

Kenapa Kamu Harus Akan Mungkin Suka Markdown?

Sudah dua bulan ini saya menggunakan Markdown, jadi alasan dibawah ini saya rangkum dari pengalaman saya.

  • Markdown lebih simple
  • Learning curve Markdown jauh lebih rendah dibandingkan HTML
  • Markdown bisa dikombinasikan dengan HTML
  • Platform agnostik. Markdown bisa diedit di semua text editor.
  • Dipakai oleh orang-orang keren di Git, reddit, dan Stack Overflow.
  • Teks menjadi mudah dibaca
